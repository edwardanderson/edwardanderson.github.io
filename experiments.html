RDFS Pydantic

Would it be useful to generate data contracts from arbitrary RDFS ontologies? RDFS Pydantic is an exploratory prototype. 

                
                    @prefix ex: <http://example.org/> .
                    @prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
                    @prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .

                    ex:Example a rdfs:Class ;
                        rdfs:label "An Example class"
                        rdfs:comment "Useful for examples" .

                    ex:name a rdf:Property ;
                        rdfs:domain ex:Example ;
                        rdfs:range rdfs:Literal

                    ex:example_property a rdf:Property ;
                        rdfs:domain ex:Example ;
                        rdfs:range ex:Example .
                
            
                
                    from pydantic import BaseModel, Field

                    class Example(BaseModel):
                        """
                        An Example class .

                        Useful for examples.
                        """
                        example_property: Example | list[Example] = Field(default_factory=list)
                        name: str | list[str] = Field(default_factory=list)

Nested Triple Trees Language

Nettle is a specification for a compact, human-friendly concrete syntax for RDF. It expresses triples as indented trees so that related resources may be visually grouped. 

                    
                    base http://example.org/
                    prefix schema https://schema.org/

                    Mick
                    a
                        schema:Person
                    schema:name
                        "Sir Michael Philip Jagger"@en
                    knows
                        Keith

Knowledge Representation Markup Language

KRML is a simple language for creating RDF knowledge graphs using Markdown elements: lists of hyperlinks, images, tables and styled or plain text.

I wrote a few words about the project on LinkedIn.

(Almost) All the Art in the Rijksmuseum

Responding to Generous Interfaces for Digital Cultural Collections, what if you could look at every single object in the Rijksmuseum? How long would that take? Scrolling by at a rate of one every fifth of a second viewing the 400,000 photographed objects would take about four hours. Or until you run out of disk space.
